# Эволюция выбора системы зрения для USV

> Документ составлен на основе 19 отчётов подбора компонентов (25.02.2026 — 11.03.2026)  
> Платформа: NVIDIA Jetson Orin Nano 8 GB  
> Бюджет: ≤ 100 000 ₽

---

## Требования к системе зрения

| Параметр | Значение |
|---|---|
| Дальность обнаружения | Объект ~3 м (лодка) на расстоянии до 100 м |
| Минимальный порог детекции | 1 м объект @ 100 м ≥ **16 пикселей** (порог YOLOv8); 3 м @ 100 м ≥ ~48–66 пикселей |
| Тип затвора | **Global Shutter** обязателен (Rolling Shutter неприемлем из-за вибрации мотора и качки) |
| Передача данных | Без потерь (RAW/YUV) или через **MJPEG**; H.264 неприемлем (межкадровые артефакты, GOP-задержка, уничтожение малых объектов) |
| Вычислительная платформа | Edge-inference на **Jetson Orin Nano 8 GB** |
| Интерфейсы подключения | USB 3.0 и/или MIPI CSI-2 (2× 22-pin порта) |
| Бюджет всей подсистемы зрения | ≤ **100 000 ₽** |
| Стереопары | Требуются для оценки расстояния до объектов |
| Защита от внешней среды | IP67 (морская среда, брызги, соль) |
| Синхронизация | Аппаратная (External Trigger / FSYNC) предпочтительна |
| FOV суммарный | > 270° (схема: борт ~120° + нос ~60° + борт ~120° = ~300°) |
| Поставщик | Только с безналичным расчётом (юридическое лицо / ВУЗ), предпочтительно склад в РФ |

**Расчёт пикселей на цель (1920×1080 или 1920×1200, объектив 8 мм, сенсор 1/2.6″):**

| Объектив | Пикс./метр @ 100 м | 1 м объект | 3 м объект |
|---|---|---|---|
| 6 мм (1920×1200, 1/2.6″) | ~17 пкс/м | 17 пкс | 51 пкс |
| 8 мм (1920×1200, 1/2.6″) | ~23 пкс/м | 23 пкс | 69 пкс |
| 8 мм (1920×1080, 1/2.6″) | ~22 пкс/м | 22 пкс | 66 пкс |

## Ключевые решения и причины отказов

### Arducam Quad-Kit AR0234 (B0495) — ОТКАЗ
- **Причина:** Продукт не существует. B0495 — одиночная USB 3.0 камера. Quad CamArray HAT доступен только для OV9281 (B0267), OV9782 (B0331), IMX219 (B0396), IMX477 (B0397), IMX519 (B0388). AR0234 Quad HAT в продаже отсутствует.

### Sony IMX477 Quad (Arducam B0397) — ОТКАЗ
- **Причина:** Rolling Shutter — критичен при вибрации мотора и качке на воде; артефакты «jelly-effect» разрушают bounding box-ы нейросети.

### Sony IMX296 (одиночный модуль) — ОТКАЗ
- **Причина:** CamArray-набор для Jetson не существует; только single-camera (Jetvariety). C-Mount крепление; поддержка Jetson требует самостоятельной адаптации драйверов; доступность в РФ низкая.

### Waveshare IMX462 — ОТКАЗ
- **Причина:** Rolling Shutter (артефакты на волнах); нет поддержки внешнего триггера (FSYNC) → аппаратная синхронизация стереопары невозможна.

### OV9281 (Arducam B0266 / B0267) — ОТКАЗ
- **Причина:** Монохромный сенсор (Ч/Б). Нет цвета — критично для распознавания буёв, судов, навигационной обстановки.

### Все GMSL2-камеры (IMX390, AR0233) — ОТКАЗ
- **IMX390 (Leopard, D3, Sensing World, TIER IV C1):** Rolling Shutter (несмотря на автомобильное позиционирование HDR 120 дБ). Десериализаторы — от $300 до $2 399. Настройка требует 2–4 недели кросс-компиляции ядра Linux. Все GMSL2-камеры с настоящим Global Shutter в бюджете проекта отсутствуют.
- **Arducam IMX477 FPD-Link III (B0524):** Rolling Shutter; максимум 2 камеры с переключением (не одновременно); требует 3 CSI-порта (на Orin Nano только 2).
- **Общий вывод:** Архитектура GMSL2 — избыточна, дорога и нереализуема на Jetson Orin Nano в рамках бюджета.

### Luxonis OAK-1 PoE (IMX378) — ОТКАЗ
- **Причина:** Rolling Shutter (артефакты на волнах); нет аппаратной синхронизации (нет M8-разъёма для FSYNC); 3 шт. ~94 230 ₽ — исчерпывает весь бюджет без резерва; доставка в РФ затруднена.

### Hikrobot MV-CU016-10GC GigE (IMX296) — ОТКАЗ от архитектуры GigE
- **Причина:** Несмотря на Global Shutter и PTP IEEE 1588, архитектура GigE признана избыточной по сложности и стоимости периферии (PoE-коммутатор, USB-GigE адаптер). IP40 — требует внешних IP67-боксов. Вся ветка GigE отклонена в пользу USB3 / MIPI CSI.

### Все Hikrobot MV-CU GigE-камеры — ОТКАЗ по интерфейсу
- **MV-CU060-10GC (IMX178):** Rolling Shutter + GigE.
- **MV-CU120-10GC (IMX226):** Rolling Shutter + GigE + избыточное разрешение (12.2 МП).
- **MV-CU050-30GC (AR0521):** Rolling Shutter + GigE.
- **MV-CU050-90GC (GMAX2505):** Global Shutter, но GigE — USB-to-Ethernet создаёт дополнительную задержку и ограничивает bandwidth.
- **Общий вывод:** Интерфейс GigE несовместим с требованием прямого подключения к Jetson Orin Nano (нет встроенного GigE порта, только USB).

### Arducam AR0234 USB 3.0 (B0495) — ОТКАЗ
- **Причина:** Недостаточная пропускная способность шины USB для 3 одновременных потоков 2.3 МП; нет аппаратной синхронизации (нет CamArray HAT для USB-версии); высокая нагрузка на CPU.

### Basler acA2440-35um — ОТКАЗ
- **Причина:** Цена ~$879–1 010/шт; три камеры ~$2 637–3 030 — вдвое превышает бюджет $1 200 на камеры. Pylon SDK работает на Jetson отлично, но цена неприемлема.

### Contrastech MARS5000S-68UM-Lite — ОТКАЗ
- **Причина:** Цена $450/шт, три камеры ~$1 350–1 608 — выходит за бюджет $1 200. SDK на ARM64 не подтверждён документально.

### MindVision MV-SUA505GC — ОТКАЗ / запасной
- **Причина:** SDK (Linux ARM Linux) заявлен вендором, но **не верифицирован** на Jetson Orin Nano. Риск несовместимости неприемлем для критичного оборудования катера.

### Ambertek SQ10 v2.0 — ОТКАЗ
- **Причина (три стоп-фактора):**
  1. **Rolling Shutter** — jelly-effect при вибрации и качке.
  2. **Нет потокового вывода** — не является UVC-камерой; видео пишется в AVI/H.264 на microSD, Jetson физически не получает кадр в реальном времени.
  3. **H.264-сжатие** — lossy-компрессия уничтожает малые объекты на фоне бликов; задержка 100–500 мс из-за буферизации GOP.

### AR0521 5МП «Global Shutter» (market-cctv.ru, kitcam.ru) — ОТКАЗ
- **Причина:** Маркетинговое упрощение. Datasheet onsemi.jp прямо указывает: «Electronic Rolling and Global Reset Release» (GRR). При GRR экспозиция начинается одновременно, но **считывание построчное** — при сильной вибрации мотора катера артефакты сохраняются. Не является настоящим Global Shutter.

### AZURE-0840CM 8 мм F4.0 — ОТКАЗ (заменён)
- **Причина:** Диафрагма F4.0 слишком тёмная для переменного освещения на воде (утро, туман, сумерки). Конструктивная виброзащита (fixed focus + fixed iris) — плюс, но компромисс по светосиле неприемлем. Заменён на CHIOPT FA0802D F1.4.

### Computar M0824-MPW2-R — ОТКАЗ
- **Причина:** Цена ~38 000–40 000 ₽ при бюджете ≤20 000 ₽. Made-to-order с неизвестным сроком поставки. Оправдан только при вибрации >3G без возможности внешней виброизоляции.

### Kowa LM8JC5MC и Kowa LM8HC-V — ОТКАЗ
- **Причина:** Kowa LM8JC5MC ~40 000+ ₽ (€406); Kowa LM8HC-V ~60 000+ ₽ (€593) — кратно превышают бюджет. Дополнительный недостаток HC-V: минимальный пиксель 5.0 мкм против 3.45 мкм у IMX264 — не раскрывает полное разрешение сенсора.

## Ключевые архитектурные повороты

### 1. Первоначальный выбор Arducam MIPI CamArray (25.02–26.02.26)

Начальная концепция: 4 камеры AR0234 в составе 2× Arducam B0492N Stereo Kit, оба CamArray HAT подключаются к обоим CSI-портам Jetson Orin Nano. Аппаратная синхронизация внутри пары через CamArray HAT; между парами — GPIO external trigger. Итоговая стоимость 70 815–73 800 ₽. Схема FOV: 120° (лево) + 60° (центр) + 120° (право) = ~300°.

**Проблема, выявленная позже:** B0492N оказался недоступен к покупке у B2B-поставщиков в РФ к марту 2026.

---

### 2. Исследование GMSL2 и отказ (27.02.26)

После подтверждения базовой MIPI-архитектуры была проведена проверка автомобильного стандарта GMSL2 как потенциально более надёжного решения. Исследованы камеры Leopard Imaging, D3 Embedded, Sensing World, TIER IV C1 — все на сенсоре IMX390. Десериализаторы: от $300 (Sensing World SG4A-NONX-G2Y-A1) до $2 399 (D3 Embedded Orin Nano 8-Cam Carrier).

**Причины отказа:**
- Все доступные GMSL2-камеры — Rolling Shutter (IMX390 — Rolling, несмотря на HDR 120 дБ).
- Установка требует кросс-компиляции ядра Linux и отладки device tree — 2–4 недели работы.
- Стоимость системы: $1 473+ за 3 камеры — в 1.5× бюджет только на камеры.
- Global Shutter GMSL2-камеры в бюджете проекта физически отсутствуют на рынке.

### 3. Исследование GigE PoE и отказ (27.02.26)

## Нерешённые технические вопросы

*(актуальные на момент финального отчёта 11.03.2026)*

1. **Интерфейс kitcam.ru AR0234 «MIPI / USB 3.0» [КРИТИЧНО]:** противоречие в спецификации (заголовок — «MIPI output», описание — «USB 3.0»). Неизвестно, есть ли реально мост MIPI→USB 3.0 на плате и поддерживается ли UVC. Необходимо запросить даташит и подтверждение UVC у поставщика до заказа.

2. **AR0234 USB 3.0 в РФ:** подтверждённых модулей AR0234 с настоящим USB 3.0 в российских B2B-магазинах по состоянию на март 2026 не найдено. Текущее решение — USB 2.0 с MJPEG.

3. **IR-фильтр для объектива 2.1 мм (боковые камеры):** объектив kit-frame 2.1 мм F1.6 не имеет встроенного IR-фильтра. Без него сенсор AR0234 даёт искажённые цвета. Решение: запросить версию с напылённым IR-фильтром 650 нм или докупить держатель с шторкой и зафиксировать в положении «День».

4. **Дисторсия объектива 2.1 мм:** широкоугольная оптика 2.1 мм даёт бочкообразную дисторсию >15–20%. Для задач оценки расстояния (depth estimation) потребуется программная калибровка (undistortion) на Jetson — дополнительная нагрузка на CPU/GPU.

5. **Pixel shift в CHIOPT FA0802D:** внутренние оптические элементы не вклеены. При ударных нагрузках >5G возможен микросдвиг. Тросовые виброизоляторы GR2-4.9D-A (этап 2) — только из AliExpress, поставка 2–4 недели, нет склада в РФ.

6. **Baseline стереопары:** расстояние между камерами B0492N в CamArray HAT явно не указано ни в одном из 19 документов — требует измерения или уточнения у Arducam.

7. **Цена VisCam NS-503UC:** mkoi.ru / cameralab.ru отвечают «по запросу»; B2B-цена при закупке 3 шт. не раскрыта. При цене >33 000 ₽/шт. система из 3 камер превысит 100 000 ₽.

8. **Синхронизация боковых AR0234 USB 2.0:** только программная (через USB), без аппаратного триггера. Точность синхронизации не верифицирована для стереоалгоритмов.

9. **Корпус IP54+ для электроники синхронизации:** рекомендован, но не выбран (упомянут в волне 4a с бюджетом 10 000–15 000 ₽).

10. **Наличие Gainta G113G-IP67:** ни у одного из трёх поставщиков нет на складе (от 4–6 недель до июня 2026). Альтернатива — корпус G107-IP67 или фрезерование из алюминиевого профиля.

## ⚠️ Противоречия между версиями

### 1. Разрешение AR0234: 1920×1200 vs 1920×1080

В разных документах разрешение AR0234 указано по-разному:
- **1920×1200 (2.3 МП)** — в файлах 01–10 (волны 1–3b), где камера используется на полном сенсоре 1/2.6″ через CamArray HAT.
- **1920×1080 (2 МП)** — в файлах 18–19 (волна 5), где камера описывается как USB 2.0 модуль kit-frame.ru/kitcam.ru.

**Объяснение:** Физически сенсор AR0234CS имеет максимальное разрешение 1920×1200 (полный кадр). USB 2.0 модули зачастую выдают кропнутый регион интереса (ROI) 1920×1080 в стандарте Full HD — либо из-за ограничений прошивки, либо для уменьшения нагрузки на USB 2.0. Это **режим кадрирования**, а не другой сенсор. Физический размер пикселя остаётся 3.0 мкм в обоих случаях.

### 3. Интерфейс kitcam AR0234 (MIPI vs USB 3.0)

Модуль kitcam.ru AR0234 описывается как «USB 3.0 WEB-камера, интерфейс MIPI» — прямое противоречие в официальной спецификации поставщика. Документ 18 (11.03.26) подробно разбирает этот парадокс:

- Наиболее вероятная причина: сенсор AR0234 имеет **нативный MIPI CSI-2 выход** (что указано в заголовке), а плата содержит **мост MIPI→USB** (Cypress FX3 или аналог), что даёт USB 3.0 на выходе.
- **Два критических риска:** (а) модуль может оказаться чистым MIPI без USB-конвертера; (б) может быть USB **2.0** (а не 3.0), что кардинально меняет пропускную способность.
- Без уточнения у поставщика (наличие моста, UVC, формат потока) — заказ категорически не рекомендован.

Одновременно в документах 01–10 Arducam AR0234 B0492N описывается как MIPI CSI-2 камера (что соответствует официальным спецификациям Arducam) — это **другая** линейка продуктов (не kitcam.ru), противоречие там отсутствует.
